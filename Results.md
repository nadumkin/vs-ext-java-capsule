# Results

Дата обновления: 2026-03-31

## Что было сделано

VS Code extension для ИИ-ассистента.

Реализованы основные части расширения:

- activity bar контейнер `Agent` и chat-панель на `WebviewViewProvider`
- базовый extension host и команды VS Code
- клиент OpenRouter с хранением API key в `VS Code Secret Storage`
- агентный цикл `LLM -> tool call -> выполнение -> LLM`
- сбор контекста активного файла
- сбор локальных импортов
- поиск связанных тестов
- передача пользовательского запроса вместе с контекстом
- инструменты для чтения, поиска, записи и замены файлов
- инструменты для запуска shell-команд и bash-скриптов
- чтение вывода управляемых shell-сессий

## Контекст, который уходит в модель

В prompt сейчас включаются:

- активный файл
- выделение в активном файле, если оно есть
- найденные импортированные локальные файлы
- связанные тесты по имени файла и класса
- пользовательский запрос

Для разрешения импортов и тестов используются безопасные эвристики на уровне workspace, без зависимости от proposed API.

## Команды и вывод в диалоге

Запускаемые `shell` и `bash` команды теперь отображаются в чате отдельными визуально выделенными блоками.

Для таких блоков показываются:

- тип команды
- сама команда или скрипт
- `cwd`
- `session id`
- `exit code`
- `stdout`
- `stderr`
- финальный статус: `RUNNING`, `OK`, `ERROR`, `TIMEOUT`, `CANCELLED`

Обновление command-блоков происходит через `upsert`, поэтому один и тот же блок обновляется на месте и не плодит дубликаты.

## Ограничения текущей версии

- `read_terminal_output` читает вывод только тех процессов, которые были запущены самим агентом
- поиск импортов и тестов построен на эвристиках, а не на полном LSP/AST-анализе
- редактирование файлов сейчас идет через запись полного содержимого, без diff preview
- вывод shell-команд теперь виден в чате по мере выполнения, но streaming ответа самой модели пока не реализован
